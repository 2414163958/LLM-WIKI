# How I use Obsidian - 摘要

本文是 [[Steph Ango]] 关于使用 [[Obsidian]] 进行知识管理的个人指南，阐述了一种自下而上、包容混乱的笔记方法。

## 核心理念

- **[[File over app]]**：数字作品应保存为可控的文件格式，而非依赖特定应用程序
- **自下而上**：通过日常积累自然涌现结构，而非预设分类体系
- **包容混乱和懒惰**：接受初期无序，通过链接和回顾逐渐形成秩序
- **链接驱动**：大量使用内部链接建立知识网络

## 主要方法要点

- **单一存储库**：避免拆分内容到多个库，保持知识完整性
- **极简文件夹**：大部分笔记放在根目录，避免分类负担
- **大量内部链接**：链接到首次提及的概念，包括未解析链接
- **一致风格**：统一的标签命名、日期格式、评分系统等规则
- **[[分形日记]]**：按日→周→月→年的层级回顾，形成可缩放的人生图景
- **随机重访**：定期随机浏览笔记，建立新链接，维护格式一致性
- **模板驱动**：每个类别都有模板，快速填充属性信息

## 关键工具和配置

- **主题**：[[Minimal]] + [[Flexoki]] 配色方案
- **同步**：Obsidian Sync（跨设备同步）
- **采集**：Obsidian Web Clipper（保存网页内容）
- **浏览**：Obsidian Bases（按类别查看笔记）
- **地图**：Obsidian Maps（地理位置相关笔记）
- **发布**：Jekyll 静态网站生成器 + Netlify 托管
- **评分系统**：1-7 分制（7=完美，1=邪恶）

## 文件夹结构

- **根目录**：个人笔记、日记、[[常青笔记]]
- **References**：外部事物（书籍、电影、地点、人物）
- **Clippings**：他人文章和剪报
- **Attachments**：图片、音频、视频、PDF
- **Daily**：每日笔记（`YYYY-MM-DD.md`）
- **Templates**：模板文件
