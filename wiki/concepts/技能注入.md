# 技能注入

## 概念

**技能注入** 是 OpenClaw 的工作区注入机制，通过在代理工作区放置特定文件，自动将知识、人格、工具定义注入代理上下文。与 [[上下文注入]] 概念紧密相关。

## 注入文件

| 文件 | 作用 |
|------|------|
| `AGENTS.md` | 代理行为指南、规则、约束 |
| `SOUL.md` | 人格定义、性格特征 |
| `TOOLS.md` | 工具使用说明、最佳实践 |

文件位于 `~/.openclaw/workspace/`，代理启动时自动读取。

## 技能组织

- **技能路径**：`~/.openclaw/workspace/skills/<skill>/SKILL.md`
- **技能类型**：bundled（捆绑）、managed（管理）、workspace（工作区）
- **安装限制**：可通过配置控制哪些技能可安装

## 与 Agent Skills 的关系

技能注入是 [[Agent Skills]] 的具体实现形式，但更强调：

1. **工作区优先**：文件在工作区，而非全局配置
2. **人格注入**：SOUL.md 提供人格层，超越纯技能
3. **动态发现**：ClawHub 支持自动搜索添加

## 与上下文注入的关系

技能注入是 [[上下文注入]] 在 AI 助手领域的应用：

- **注入时机**：代理启动时自动注入
- **注入内容**：行为规则、人格、工具说明
- **注入效果**：代理获得工作区特定的知识和能力

## 配置示例

```json
{
  "agents": {
    "defaults": {
      "workspace": "~/.openclaw/workspace"
    }
  }
}
```
