# 验证循环

AI 辅助开发中的持续验证机制，确保生成代码和决策的质量。

## 核心思想

AI 生成的代码和决策需要系统性验证，而非一次性通过。验证循环提供**检查点 → 评估 → 修正 → 再验证**的闭环。

## Everything Claude Code 的实现

### 两种验证模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **检查点** | 关键节点保存状态，失败可回滚 | 长任务 |
| **持续评估** | 每步实时评估 | 短任务 |

### 评估器类型

| 类型 | 功能 |
|------|------|
| 单元测试评估器 | 运行测试套件 |
| 类型检查评估器 | TypeScript/静态类型检查 |
| Lint 评估器 | 代码风格和规范检查 |
| 安全评估器 | [[AgentShield]] 漏洞扫描 |
| 自定义评估器 | 项目特定验证逻辑 |

### pass@k 指标

多次尝试中通过验证的概率：

- **pass@1**：一次尝试通过率
- **pass@k**：k 次尝试中至少一次通过率

用于量化 AI 生成质量。

## 命令体系

| 命令 | 功能 |
|------|------|
| `/checkpoint` | 保存验证状态 |
| `/verify` | 运行验证循环 |
| `/eval` | 按标准评估 |
| `/test-coverage` | 测试覆盖率分析 |

## 与 TDD 的关系

验证循环是 TDD 的自动化增强：

| 维度 | TDD | 验证循环 |
|------|-----|----------|
| 验证时机 | 手动触发 | 自动/持续 |
| 验证范围 | 测试 | 测试 + 类型 + Lint + 安全 |
| 回滚能力 | 无 | 检查点回滚 |
