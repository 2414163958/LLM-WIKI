# 多智能体协作

多个 AI 模型/智能体并行工作、各司其职的协作模式，由 [[oh-my-openagent]] 实现。

## 核心理念

不依赖单一模型，而是运营一个联合体：Claude 做编排、GPT 做推理、Kimi 提速度、Gemini 处理视觉。模型越来越便宜，没有提供商垄断。

## 智能体角色

### Sisyphus（主指挥官）

- 制定计划、分配任务给专家团队
- 激进并行策略推动任务直至完成
- 从不半途而废
- 模型：claude-opus-4-6 / kimi-k2.5 / glm-5

### Hephaestus（自主深度工作者）

- 只给目标，不给具体做法
- 自动探索代码库模式
- 从头到尾独立执行任务
- 不中途要保姆式指导
- 模型：gpt-5.4

### Prometheus（战略规划师）

- 访谈模式：动代码前先采访用户
- 深挖需求、指出模糊地带
- 产出严密论证的计划
- 模型：claude-opus-4-6 / kimi-k2.5 / glm-5

## 调度机制

智能体选择**类别（Category）**而非具体模型：

| 类别 | 作用领域 |
|------|----------|
| `visual-engineering` | 前端、UI/UX、设计 |
| `deep` | 深度自主调研与执行 |
| `quick` | 单文件修改、修错字 |
| `ultrabrain` | 复杂硬核逻辑、架构决策 |

系统自动将类别映射到最合适的模型。

## 并行工作

- 后台智能体：同时发射 5+ 个专家并行工作
- 保持上下文干净，随时获取成果

## ECC 的 multi-* 命令

[[Everything Claude Code]] 提供类似的多智能体编排能力：

| 命令 | 功能 |
|------|------|
| `/multi-plan` | 多智能体任务拆解 |
| `/multi-execute` | 多智能体工作流编排 |
| `/multi-backend` | 后端多服务编排 |
| `/multi-frontend` | 前端多服务编排 |
| `/multi-workflow` | 通用多服务工作流 |

需要 ccg-workflow 运行时支持。

## Superpowers 的子代理驱动开发

[[Superpowers]] 的 **subagent-driven-development** 技能提供类似能力：

- 为每个任务派遣新的子代理
- 两阶段审查：先检查规范符合性，再检查代码质量
- 代理可自主工作数小时不偏离计划
