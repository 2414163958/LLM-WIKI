# File over app

应用程序会消亡，但文件格式可以持久。如果你想创建能够永久保存的数字作品，这些文件必须是你可以控制的，并且格式易于检索和读取。

## 核心原则

- **数据主权**：你拥有自己的数据，而不是被锁定在某个应用程序中
- **格式开放**：使用开放、标准化的格式（如 Markdown、纯文本）
- **长期可读**：确保未来几十年后仍能访问和读取这些文件
- **迁移自由**：随时可以切换工具，而不丢失任何内容

## 实践方法

- 选择使用开放文件格式的工具（如 Obsidian、Logseq 使用 Markdown）
- 避免依赖专有格式或云端锁定的应用
- 定期备份文件到本地存储
- 使用版本控制（如 Git）追踪文件变化
- 文件命名和组织遵循一致的规范
- 将文件存储在你完全控制的目录中
