# 设备节点

## 概念

**设备节点** 是 OpenClaw 的分布式执行架构。iOS、Android、macOS 设备作为节点，通过 WebSocket 与 Gateway 配对，执行设备本地操作（相机、屏幕录制、通知等）。主机执行工具，设备执行设备本地操作。

## 节点类型

| 节点 | 功能 | 配对方式 |
|------|------|----------|
| iOS node | Canvas、语音唤醒、通话模式、相机、屏幕录制 | Bonjour + 设备配对 |
| Android node | Canvas、相机、屏幕录制、设备命令（通知/位置/短信/照片/联系人/日历） | 设置代码/手动配对 |
| macOS node | system.run/notify、canvas/camera | macOS app 节点模式 |

## 操作路由

- **Gateway 主机**：默认执行工具、通道连接
- **设备节点**：通过 `node.invoke` 执行设备本地操作

区分原则：exec 运行在网关所在位置，设备操作运行在设备所在位置。

## 节点协议

节点通过 Gateway WebSocket 发布功能和权限映射：

- `node.list` — 发现可用节点
- `node.describe` — 描述节点功能和权限
- `node.invoke` — 执行节点操作

## 权限控制（macOS）

macOS TCC 权限通过 Gateway 协议管理：

- `system.run` — 运行本地命令，返回 stdout/stderr/exit code
- `system.notify` — 发送用户通知
- `needsScreenRecording` — 设置需要屏幕录制权限
- `PERMISSION_MISSING` — 权限缺失时的错误返回

## 远程网关场景

Gateway 可在 Linux 实例运行，设备节点通过网络配对：

- Gateway 处理工具执行和通道连接
- 设备节点处理本地操作（通知、相机等）
- 网络连接通过 Tailscale 或 SSH 隧道

## 与 MCP 的关系

设备节点是 [[MCP]] 工具协议的具体应用场景：

- MCP 定义工具接口标准
- 设备节点是工具在移动端的具体实现
- 通过 Gateway WebSocket 路由调用
