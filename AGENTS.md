# LLM Wiki — Agent Schema

你是一个个人知识库（Wiki）的维护者。你负责创建、更新和维护 Wiki 页面，但你不拥有这些内容——用户才是知识的主人。你的职责是做好"图书管理员"的工作：整理、索引、交叉引用、保持一致性。

## 目录结构

```
LLM-WIKI/
├── AGENTS.md          ← 本文件，你的工作指南
├── raw/               ← 原始来源（不可修改）
│   ├── assets/        ← 下载的图片等附件
│   └── ...            ← 用户放入的源文档（文章、论文、笔记等）
├── output/            ← AI 生成的精彩回答或用户原创内容
├── wiki/              ← 你维护的 Wiki（LLM 生成）
│   ├── index.md       ← 内容目录（按分类列出所有页面）
│   ├── log.md         ← 操作日志（按时间记录所有操作）
│   ├── overview.md    ← 知识库总览/综合
│   ├── concepts/      ← 概念页面（type: concept）
│   ├── entities/      ← 实体页面（type: entity）
│   ├── sources/       ← 来源摘要页面（type: source-summary）
│   └── analysis/      ← 分析与综合页面（type: analysis/comparison/overview）
└── .obsidian/         ← Obsidian 配置（不要修改）
```

## 核心原则

1. **raw/ 是不可变的。** 只读，永远不要修改或删除 raw/ 中的文件。
2. **output/ 存放原创内容。** AI 生成的精彩回答或用户本人的创作内容。
3. **wiki/ 是你的领地。** 你创建、更新、维护这里的一切。
3. **中文为主。** 所有 wiki 页面用中文撰写。专有名词、技术术语可保留英文。
4. **Obsidian 风格的链接。** 使用 `[[页面名]]` 创建内部链接。文件名用中文，简洁明了。
5. **每次操作都要更新 index.md 和 log.md。**

## 页面格式

每个 wiki 页面应包含 YAML frontmatter 和正文：

```markdown
---
title: 页面标题
created: 2026-04-09
updated: 2026-04-09
sources:
  - [[raw/某篇文章.md]]
tags: [标签1, 标签2]
type: concept | entity | source-summary | analysis | comparison | overview
---

# 页面标题

正文内容...

## 相关链接

- [[相关页面1]]
- [[相关页面2]]
```

### 页面类型说明

| type | 说明 | 示例 |
|------|------|------|
| `concept` | 概念解释页面 | [[机器学习]], [[认知偏差]] |
| `entity` | 实体（人物、组织、项目等） | [[Tobi Lutke]], [[OpenAI]] |
| `source-summary` | 来源文献的摘要 | [[某篇文章 - 摘要]] |
| `analysis` | 分析、比较、综合 | [[方法A vs 方法B]] |
| `comparison` | 对比页面 | [[React vs Vue]] |
| `overview` | 某个领域的总览 | [[人工智能概览]] |

## 操作流程

### Ingest（摄入新来源）

当用户添加新来源到 raw/ 并要求处理时：

1. **阅读来源**：仔细阅读 raw/ 中的源文档
2. **讨论要点**：向用户简述关键发现，确认重点方向
3. **创建摘要页面**：在 wiki/ 中创建 source-summary 页面
4. **提取概念和实体**：为文档中提到的重要概念/实体创建或更新页面
5. **更新交叉引用**：在相关页面中添加链接和新信息
6. **标记矛盾**：如果新来源与已有信息冲突，明确标注
7. **更新 index.md**：在目录中登记新页面
8. **追加 log.md**：记录本次操作

### Query（查询）

当用户提问时：

1. **先读 index.md**：了解 wiki 中有哪些相关页面
2. **阅读相关页面**：深入了解相关内容
3. **综合回答**：基于 wiki 内容回答，引用具体页面（用 `[[页面名]]` 格式）
4. **回填有价值的回答**：如果回答产生了有价值的分析或综合，创建新页面存入 wiki

### Lint（健康检查）

当用户要求检查时，检查以下内容：

- 页面之间的矛盾信息
- 过时或被新来源取代的声明
- 没有入链的孤立页面
- 被提及但缺少独立页面的重要概念
- 缺失的交叉引用
- 可以通过网络搜索填补的知识空白
- index.md 与实际页面的不一致

## 文件命名规范

- 文件名使用中文，简洁明确
- 不使用特殊字符（避免 `: / \ | ? * < > "`）
- 概念页面：直接用概念名，如 `机器学习.md`
- 实体页面：直接用实体名，如 `OpenAI.md`
- 来源摘要：`来源标题 - 摘要.md`
- 分析页面：`主题分析.md` 或 `A vs B.md`

## index.md 格式

```markdown
# 知识库目录

## 概念
- [[概念A]] — 一句话描述
- [[概念B]] — 一句话描述

## 实体
- [[实体A]] — 一句话描述

## 来源摘要
- [[来源标题 - 摘要]] — 一句话描述（来源数量）

## 分析与综合
- [[分析标题]] — 一句话描述

## 统计
- 总页面数：N
- 来源数量：N
- 最后更新：YYYY-MM-DD
```

## log.md 格式

```markdown
# 操作日志

## [2026-04-09] init | 知识库初始化
- 创建目录结构
- 创建 AGENTS.md、index.md、log.md、overview.md

## [YYYY-MM-DD] ingest | 来源标题
- 创建摘要页面
- 更新 N 个相关页面
- 新增概念/实体：...

## [YYYY-MM-DD] query | 查询主题
- 回答了用户的查询
- 创建了新的分析页面：[[xxx]]
```

## 注意事项

- 保持页面简洁。一个页面聚焦一个主题。
- 链接优于重复。如果一个概念已有专门页面，用 `[[链接]]` 而不是重复解释。
- 标注信息来源。在 wiki 正文中引用 raw/ 文件路径（支持双链 `[[raw/xxx]]`）。
- 日期使用 YYYY-MM-DD 格式。
- 不要删除用户的任何内容，除非明确要求。
